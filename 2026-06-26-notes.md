## Gotchas

- **`set_defaults` overrides constructor-supplied attributes on `.new()`**: `after_initialize set_defaults if: :new_record?` runs AFTER the constructor sets attributes, so `NotificationSetting.new(user: u, webhook_url: "x")` ends up with `webhook_url == nil`. Worth pinning because a future "fix" would silently change persistence behavior.
- **`set_defaults` does NOT re-fire on update**: only `if: :new_record?`. Use `update!` (not `.new(attrs)`) when testing the `cast_string_value_to_boolean` callback.
- **`NotificationSetting::DEFAULT_SETTING` is frozen**, includes 19 keys (6 categories × 3 channels + `webhook_url`).
- **`article_bought_daily_times` is an exposed store accessor with NO default value** (it's not in DEFAULT_SETTING). Both `article_bought` and `article_rewarded` declare `daily_times` as accessors but only `article_bought_daily_times` is actually exercised by the code.
- **`cast_string_value_to_boolean` covers 18 accessor columns** (`<category>_<channel>` for 6×3) plus nil-safe casts.
