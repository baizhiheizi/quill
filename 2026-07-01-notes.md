## Gotchas

- **Article `cannot_edit_frozen_attributes_once_published`** fires on first save when `published_at` is set in the constructor. Workaround: create with `state: :drafted`, then `article.publish!` via AASM.
- **`Article` requires `content`**: `has_rich_text :content` triggers `ensure_content_valid` on the `publish` event. Set `article.content = "<p>test</p>"` before `save!` when authoring state is not :drafted.
- **`Noticed::Event.type` is an STI column.** Use a real notifier class name (`TransferProcessedNotifier`) in tests — fake names raise `ActiveRecord::SubclassNotFound` on `.type` lookup.
- **`noticed_notifications` doesn't denormalise `record`.** To count notifications: `Noticed::Notification.joins(:event).where(noticed_events: { record_type: "Tagging", record_id: id })`.
- **`Tagging#notify_subscribers`** is on `after_create_commit`, but it's also triggered by explicit `tagging.send(:notify_subscribers)`. Notifier serialises params (incl. `tagging`), so the tagging MUST be saved before invoking `notify_subscribers` directly. Use `Tagging.create!` not `Tagging.new` in manual-invocation tests.
- **`User#blocked_user_ids_relation`** = users I have blocked (not users who blocked me). `User.where.not(id: author.blocked_user_ids_relation)` filters out users the AUTHOR has blocked.
