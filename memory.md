- **Stimulus dead-code detection**: scan `data-controller="xxx"`, `data-controller='xxx'`, `controller: 'xxx'`, `element.dataset["controller"] = "xxx"`. Use `re.findall(r'data-controller=["\']([^"\']+)["\']', content)`. An identifier is "dead" only if (a) no view references it AND (b) the controller file does not exist.
- **Empty controller stub detection**: a Stimulus controller with only a `connect()` that does `console.log(...)` is a stub left from a migration.
- `bin/ci` uses `ActiveSupport::ContinuousIntegration` with: `Setup`, `Style: Ruby`, `Style: JavaScript`, `Tests: Rails`, `Tests: Seeds`.
- `Action` extends `ActiveRecord::Base` directly (not `ApplicationRecord`), polymorphic `target`/`user` with `optional: true`, has `after_create :notify_target` + `before_destroy :destroy_notifications`.
- `config/routes.rb` includes a domain redirect for `prsdigg.com|bunshow.jp` → `https://quill.im/$path` at the top; preserve when refactoring routes.
- For JS-only changes, the local-toolchain equivalent of `bun run lint-check` is `node_modules/.bin/prettier --check app/javascript/<changed-file>`; `node esbuild.config.js` produces the same bundle CI builds.
- **Dead Rails action detection**: for any `def <verb>` on a controller, confirm the verb is exposed by the matching route declaration AND a corresponding view file exists at `app/views/<namespace>/<controller>/<verb>.html.erb`. If both fail, the action is dead.
- **Stale Ruby idiom**: `params[:foo]&.present?` can be `params[:foo].present?` if the route never receives `nil` for that key.
