- `safeoutputs create_pull_request` returns the patch path and bundle size; the actual PR number is assigned by the workflow runner, so PR-intent items in Suggested Actions reference the branch + commit + patch path rather than a PR number.
- `gh aw compile` requires at least one line of body content (it errors with "no markdown content found" when the body is empty), even with a `source:` line that points to an upstream workflow.
- `update_issue` body size limit is 10 KB; this run's body is ~7.8 KB.
- Notifier test setup gotchas worth remembering:
  - `PreOrder#setup_attributes` regenerates `follow_id` (and `trace_id` for some order_types) on create. For tests that need a specific `follow_id`, create the PreOrder, read back the regenerated `follow_id`, then `update_columns(trace_id:, follow_id:)` to patch.
  - `Payment#setup_attributes` runs as `before_validation :setup_attributes, on: :create` and assigns `amount`, `memo`, `asset_id`, `opponent_id`, `snapshot_id`, `trace_id`, `payer` from `raw`. With `validate: false`, that callback is skipped, so tests must assign those columns explicitly.
  - `Payment#pre_order` memoizes `@pre_order ||=`, so if `setup_attributes` ran (in the validation path) and `pre_order` was nil at that moment, subsequent calls still return nil. Reading back from a fresh `payment.decoded_memo["f"]` after creating the PreOrder avoids the issue when validation is skipped.
  - `SwapOrder` belongs_to `payment` (required), so `update!(state: :refunded)` fails validation in a test that doesn't build a real Payment. Use `update_columns(state: "refunded" / "rejected")` to bypass validation and exercise the message branch logic.
  - `User#messenger?` is not a column — it reads `authorization.provider == "mixin"`. To test "not a messenger recipient" paths, stub `user.define_singleton_method(:messenger?) { false }`.
  - `Action` has `after_create :notify_target` which calls `SubscribeUserActionCreatedNotifier.with(...).deliver(target)` for subscribe actions. Stub this with `define_singleton_method(:notify_target) { }` in test setup to avoid double-delivery.
