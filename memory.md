## Notes

- **`bundle outdated` and `dependabot alerts` MCP are NOT reliable signals in this sandbox.** `bundle outdated` errors on read-only filesystem trying to refetch git-sourced gems; `dependabot alerts` errors on pagination param. Treat both as inconclusive.
- **`bin/rubocop` does NOT lint ERB or YAML.** `bin/rails zeitwerk:check` is the local CI load-path check.
- **Stimulus dead-code detection**: scan `data-controller="xxx"`, `data-controller='xxx'`, `controller: 'xxx'`. "Dead" only if (a) no view references it AND (b) the controller file does not exist.
- **Empty controller stub detection**: a Stimulus controller with only a `connect()` that does `console.log(...)` is a migration stub.
- **Dead Rails action detection**: for any `def <verb>` on a controller, confirm the verb is exposed by the matching route AND a view file exists.
- **Admin filter typo pattern**: `where(col_a: params[:col_b], col_b: params[:col_b])`.
- **Pre-existing CI-clean test errors**: `test/controllers/articles_controller_test.rb` (3 errors, asset pipeline) and `test/controllers/pre_orders_controller_test.rb` (2+ errors, SSL to Mixpay). CI stubs/supplies them.
- **`remote_image_tag`** (`ef58c599`): `nil` for nil/blank or non-absolute-URL; otherwise `image_tag`. `lazy: true` → `loading: "lazy"`.
- **`User#available_articles` perf**: push `.uniq` to SQL via `.or.or.distinct`.
- **`includes(:currency).sum(...)` antipattern**: use `joins(:currency)`. **Closed in `Article#author_revenue_usd` / `#reader_revenue_usd`**.
- **`HomeController#active_authors` perf**: PR #1735 uses `NOT IN (SELECT ...)`.
- **`setTimeout(fn(), ms)` antipattern**: invokes `fn()` synchronously. Fixed in `cbd10abd`.
- **`TokenAsset`** (PORO at `app/models/token_asset.rb`): Balance = `token["balance"].to_r / 10**token["decimals"].to_i` (round 8 dp), or 0 if `token.blank?`. Delegates 9 accessors to currency. `attr_reader :contract` declared but `@contract` never assigned — returns `nil`.
- **`Currency#save` raises in test env** because `before_validation :set_defaults` calls `QuillBot.api.asset(asset_id)`. Tests use `Currency.new(price_usd: ...)` in memory.
- `Settings.twitter_account` is `prsdigg` in `config/settings/test.yml`. `QuillBotStub#with_quill_bot_stub` provides a `FakeApi` with `FAKE_CLIENT_ID`.
- `config/routes.rb` includes a domain redirect for `prsdigg.com|bunshow.jp` → `https://quill.im/$path` at the top; preserve when refactoring.
- `ERB::Util.url_decode` does not exist in Rails 8.1; use `CGI.unescape`.
- **`CreateTagService` subtle behavior**: `tag_names.blank?` short-circuits only for `nil` and `[]`. Non-empty array of blanks gets `compact_blank!` cleanup.
- **`NotifierHelpers#deliver_notifier!`** signature: `deliver_notifier!(notifier_class, record:, recipient:, **params)`.
- **`safeoutputs create_pull_request` returns success but the branch may not reach `origin` immediately**: confirmed 2026-06-28 and 2026-06-29. The push happens out-of-band. Maintainer should pull the patch/bundle from `/tmp/gh-aw/aw-<branch-slug>.{patch,bundle}` only if no PR appears within a day.
- **Concern testing pattern**: write the test through one canonical instance (e.g. `Comment` for `SoftDeletable`); add aliased-scopes instances where needed. Avoid mocking the callback chain.
- **Order uniqueness validation workaround**: `Order`'s `uniqueness` on `(order_type, buyer_id, item_id, item_type)` for buy_article blocks multiple buys on the same `(buyer, article)` pair. Use `save!(validate: false)` via a `create_buy_order_raw!` helper when exercising early-order branches.
- **`test/models/concerns/` directory** has 7 test files. Per #1744, 5 of 12 model concerns remain untested.
- **`Nokogiri::HTML.fragment` iteration**: `fragment.children` returns `Node` instances (Element, Text, Comment) — NEVER `NodeSet`. Any `case child when Nokogiri::XML::NodeSet` branch in code that iterates `.children` is dead code.
- **Test env cache pattern**: `Rails.cache = ActiveSupport::Cache::MemoryStore.new` in setup, restore `@previous_cache` in teardown.
- **AR association re-query on update**: `article.update!(association: x)`, `article.update_columns(association_id: ...)`, and even `article.association` re-queries the DB and replaces the in-memory object. Singleton-method stubs on the original object are lost. Use a real persisted record + the existing helper (`create_buy_order!` for `buy_article`), or skip the branch.
- **Service stub restore pattern**: For tests that pin delegation contracts (e.g. `Order#distribute! → Orders::DistributeService.call`), monkey-stub the service method and restore in `ensure`. Use `Orders::DistributeService.singleton_class.send(:remove_method, :call)` then `define_method(:call) { |order| new(order).call }` to restore the original.
