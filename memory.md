## Notes

- `gh aw compile` produces `.patch` + `.bundle` artefact under `/tmp/gh-aw/`; PR finalised via `safeoutputs create_pull_request`.

- `bin/rubocop` does NOT lint ERB or YAML. `bin/rails zeitwerk:check` is the local equivalent of the CI load-path check.
- For JS-only changes, the local equivalent of `bun run lint-check` is `node_modules/.bin/prettier --check app/javascript/<changed-file>`.
- **Stimulus dead-code detection**: scan `data-controller="xxx"`, `data-controller='xxx'`, `controller: 'xxx'`, `element.dataset["controller"] = "xxx"`. An identifier is "dead" only if (a) no view references it AND (b) the controller file does not exist.
- **Empty controller stub detection**: a Stimulus controller with only a `connect()` that does `console.log(...)` is a stub left from a migration.
- **Dead Rails action detection**: for any `def <verb>` on a controller, confirm the verb is exposed by the matching route declaration AND a corresponding view file exists at `app/views/<namespace>/<controller>/<verb>.html.erb`.
- **Admin filter typo pattern**: `where(col_a: params[:col_b], col_b: params[:col_b])` — copy-paste where the second binding uses the wrong param key. `grep -rn "params\[:[a-z_]*\]\.\?$" app/controllers/admin/` if extending.
