- **Unknown / missing `user_id`**: returns `{ approved: false }` (JSON shape verified).

## Gotchas (new this run)

- **`Payment#create!` auto-transitions to `completed` via `after_create :generate_order!` → `place_article_order!` → `complete!`**. So the default state for a freshly created payment is "completed", NOT the AASM initial "paid". To create a payment in a different state (paid / refunded), either:
  1. Stub `generate_order!` on the instance before `save!` to stop the auto-transition, then use `update_columns(state:)` to set the desired value (AASM bypass — safe for the controller's filter since it only reads the column).
  2. Use a memo type that doesn't match `memo_correct?` (e.g., empty `t` key) so `generate_order!` early-returns.
- **`Article.create!(state: "published", published_at: 2.weeks.ago)` raises "Content can't be blank, Asset cannot change"** because `validate_rich_text_content_presence?` returns true for non-drafted articles. The right pattern: `Article.create!(state: "drafted", ...)` then `article.content = "..."` + `article.published_at = ...` + `article.publish!` (AASM). The AASM `publish` event runs `ensure_content_valid` which requires `content.present?`.
- **`APIController` test pattern**: `get path, as: :json` + `assert_equal({...}, response.parsed_body)` works without any auth header. `IntegrationTestCase` is the right base class.
