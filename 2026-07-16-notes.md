## Gotchas (new this run)

- **`QuillBot.api.client_id` returns `nil` in the test environment** unless wrapped in `with_quill_bot_stub`. Without the stub, `Splitter#collect_assets` silently produces zero transfers because `Transfer#ensure_opponent_presence` rejects the `opponent_id: nil` row. Discovered after the first test pass with zero transfers ‚Äî the production code looks like it ran, but every `transfers.create` raised a validation error and the loop just continued. Every `collect_assets` invocation now wraps in `with_quill_bot_stub`. Generalises: any production code path that reaches `QuillBot.api.client_id` in a test will silently break unless `with_quill_bot_stub` is in scope.
- **`Transfer#unprocessed` scope already excludes stale transfers** (`where(processed_at: nil).where(stale_at: nil)`), so an initial "stale transfer also blocks creation" test was wrong ‚Äî it actually allows re-creation. Removed the stale test; the unprocessed test covers the path the production code actually checks.
