## Gotchas (new this run)

- **`before_validation on: :create` fires on every `valid?` call** — validation tests need `define_singleton_method(:setup_attributes) { }` on the instance when leaving `raw` nil to exercise the raw-presence validation. Generalises to any model with a `before_validation` that reads the column being validated.
- **Mocha's `.stubs` is NOT available** — only `define_singleton_method` works. Critical detail: `belongs_to` returns a fresh instance, so `users(:author).define_singleton_method(:notify_for_login) { ... }` does NOT propagate to `msg.user`. Stub on the actual association instance returned by `msg.user`. This was the cause of two test failures in the prior run.
- **`enqueued_jobs.size` after `create!` includes the after_commit enqueue.** The `process_async` enqueue test asserts `2` total (one from `after_commit`, one from the explicit call) and checks `enqueued_jobs.last` is `MixinMessages::ProcessJob`. Earlier I tried asserting `== 1` and got `2`.
- **`safeoutputs create_pull_request` returns `patch` and `bundle` paths** — these are stored for the framework to apply at workflow completion, not directly creating the PR. The PR appears in the repo after the workflow exits. To reference the new PR in the Monthly Activity issue, point to the branch + commit + run URL, not a PR number (which doesn't exist yet at issue-update time).

## Side-findings (out of scope, for Maintainer)

- **`MixinMessage#touch_proccessed_at` misspelled** in production (`processed` is the intended spelling). Renaming would orphan callers; out-of-scope test PR change.
- **`MixinMessage#setup_attributes` does not nil-guard `raw`** — `raw["data"]` raises `NoMethodError` on `raw == nil`. Same hazard as `MixinNetworkUser#avatar`. A defensive `return unless raw.is_a?(Hash)` would close it.
- **`process_user_message` calls `QuillBot.api.unique_uuid(user_id)` without a rescue path** — `MixinMessages::ProcessJob` is `queue_as :critical` and retries forever on raise. Worth a future rescue-and-log PR.
