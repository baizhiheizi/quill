## Gotchas (new this run)

- **`MixinMessage#setup_attributes` dereferences `raw["data"]`** — any test that intentionally leaves `raw` nil (to exercise the `raw` presence validation) needs `define_singleton_method(:setup_attributes) { }` on the instance. Generalises to any model with a `before_validation` callback that reads from the column being validated.
- **Mocha's `User.any_instance.stubs(...)` is NOT available** in this repo. Only `define_singleton_method` works. Capture state in a closure or local variable (e.g. `called = false; user.define_singleton_method(:notify_for_login) { called = true }`).
- **`belongs_to :user, primary_key: :mixin_uuid`** — `user_id` is a UUID, not a numeric ID. `msg.user` is nil for unknown UUIDs.
- **`SKIP_CSS_BUILD=1 bin/rails test:models`** bypasses `test:prepare`'s Bun CSS bundling step. `bin/rails test test/models/foo_test.rb` direct file-level runs already work without this env var (only `bin/rails test`/`test:models` global aliases trigger prepare). Discovered when `bin/rails test:models` errored with `cssbundling-rails: No suitable tool found`.
- **`update_issue` body limit is also 10 KB** (not just `create_pull_request`'s). Monthly Activity issue body must stay under 10 KB. Trim older Run History entries to 2-3 lines each to fit.

## Side-findings

- **`MixinMessage#touch_proccessed_at` misspelled** in production (`processed` is the intended spelling). Fixing the name would orphan callers; out-of-scope test PR change.
- **`MixinMessage#setup_attributes` doesn't guard against `raw == nil`** — same nil-`raw` hazard as `MixinNetworkUser#avatar`. A defensive `return unless raw.is_a?(Hash)` would close it.
- **`process_user_message` calls `QuillBot.api.unique_uuid(user_id)` without a rescue path** — `MixinMessages::ProcessJob` is `queue_as :critical` (retries forever on raise). Worth a future rescue-and-log PR.
