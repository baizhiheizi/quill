## Catalog

Quill ships 46 Stimulus controllers, registered by `app/javascript/controllers/index.js` (the file is auto-generated by `./bin/rails stimulus:manifest:update`). They are grouped below by what they cover. Each identifier maps to `app/javascript/controllers/<snake_case>_controller.js` and is mounted in views via `data-controller="<kebab-id>"`.

### Forms and editing

| Identifier | Purpose |
|---|---|
| `article-form` | Large article editor controller. Debounced autosave and submit, modal hookup, currency/asset picker, revenue-ratio inputs, intro character counter. |
| `autosave` | Debounced form `requestSubmit()` once `delay` value passes 0. Skips if `window._rails_loaded` is unset. |
| `nested-form` | Stimulus wrapper for the cocoon-style nested-fields pattern. |
| `comment-form` | Comment composer. |
| `collections-form-component` | Collections edit form component. |
| `pre-orders-form-component`, `pre-orders-payment-component`, `pre-orders-state-component`, `pre-orders-mixpay-button-component`, `pre-orders-pay-button-component` | Components around the pre-orders purchase flow. |
| `tags-select` | Tag selector. |
| `references-select` | Cross-article reference selector. |
| `select-currency`, `select-menu` | Generic pickers. |
| `textarea-autogrow` | Auto-resize textarea (input + resize listeners; debounced). |
| `preview-upload` | Image-preview before upload. |

### Pagination and scrolling

| Identifier | Purpose |
|---|---|
| `infinite-scroll` | `IntersectionObserver` on `scrollArea` triggers `get(next.href, { responseKind: "turbo-stream" })` against `pagination`'s first anchor. |
| `load-more` | Click-driven equivalent: fetches the URL in `event.params.link` as a turbo-stream. |
| `prefetch` | Link prefetch on hover/intersection. |
| `scroll-to` | Smooth-scroll to a target. |
| `floating` | Mobile floating action bar. Document-level `{ passive: true }` scroll listener; debounced show / hide; `disconnect()` removes the stored listener reference. |
| `infinite-scroll` and `load-more` are paired: views that render as Turbo Streams use `load-more` (button), views that scroll-paginate use `infinite-scroll` (auto). |

## Patterns to know

**Debounce via `underscore`.** Controllers that respond to high-frequency events (`search`, `autosave`, `article-form`, `floating`, `textarea-autogrow`) wrap their handler with `debounce(fn, ms)` in `connect()` or `initialize()`. Wrap the function, not its return value — `debounce(classList.add, 1000)` invokes `classList.add` immediately because the return value of `classList.add` is `undefined`.

**Lifecycle cleanup.** `disconnect()` must release any listener the controller added so Turbo reconnects don't accumulate handlers. `floating` and `textarea-autogrow` are the templates: store the bound handler in `this`, then remove it from the same target in `disconnect()`. The same discipline applies to `IntersectionObserver`s and `setTimeout` handles — `flash` clears its `this.timeout` in both `hide()` and `disconnect()`.

**Turbo-Stream fetches.** Controllers that mutate server state without a full page reload import `get`/`put` from `@rails/request.js` and pass `responseKind: "turbo-stream"`. Examples: `infinite-scroll`, `load-more`, `turbo`, `search`. Pair this with `contentType: "application/json"` when the endpoint expects JSON.

**`useTransition` from `stimulus-use`.** `flash` uses `useTransition(this)` to get `enter()` / `leave()` helpers, then awaits `leave()` before removing the DOM node.

**Theme reload hook.** `darkmode` calls `reloadTheme()` from `mixin-messenger-utils` after each toggle so the Mixin messenger iframe stays in sync.
