## Gotchas

- **`Transfer#process_safe_transfer!` calls `check!` first**, which sets `processed_at` via `QuillBot.api.safe_transaction` on success. So `create_safe_transfer` is only invoked when `safe_transaction` raises `MixinBot::NotFoundError`. Test both branches separately.
- **`Bonus` model has table_name mismatch**: `Bonus.table_name` returns `"bonus"` (singular) but the migration creates `"bonuses"`. The existing `bonus_test.rb` is empty (only commented truth test). Skipped the Bonus branch inline with comment; fixing `self.table_name = "bonuses"` is a separate bug fix.
- **`belongs_to :recipient, primary_key: :mixin_uuid, foreign_key: :opponent_id`** loads a fresh User instance ‚Äî `define_singleton_method` on a fixture instance does NOT propagate. Use `UserAuthorization.define_method(:has_safe?) { ... }` (class-level) instead. Use `ensure` to restore.
- **`UserAuthorization#has_safe?`**: `raw["has_safe"].present?` returns false for BOTH `nil` AND `false`, so the "false" branch always falls through to `refresh!` which makes a real API call. Cannot test "false" via `raw` updates alone ‚Äî must stub `has_safe?` or `refresh!`.
- **`process!` source dispatch**: Payment branch calls `payment.refund! if payment.may_refund?`; Bonus branch calls `bonus.complete! if bonus.may_complete?`. Payment must be in `:paid` state for `may_refund?` to be true ‚Äî `update_column(:state, "paid")` after `create_payment!`.
- **`safeoutputs create_pull_request` 10 KB limit** on body ‚Äî Monthly Activity issue body must stay under 10 KB. Truncate older run history entries (one-line summaries) when adding new entries.
