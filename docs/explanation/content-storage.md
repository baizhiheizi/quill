# Content storage

> **30-second summary:** Published article content lives inside Quill: current formatted bodies in **ActionText**, legacy Markdown fallback in `articles.legacy_markdown_content`, history in **`ArticleSnapshot`** JSON, and article images/posters/covers in **ActiveStorage**. Quill does **not** mirror content to any external permanence network; the application is the source of truth.

## Why this page exists

Quill once uploaded published content to [Arweave](https://www.arweave.org/) and included a [Mirror.xyz](https://mirror.xyz/) import flow. Both were removed in #1582 and formalized by [`openspec/specs/content-storage/spec.md`](../../openspec/specs/content-storage/spec.md), which defines the `content-storage` capability and its out-of-scope guarantees.

## What is stored where

| Asset | Lives in | Notes |
|-------|----------|-------|
| Current formatted article body | `action_text_rich_texts` (ActionText) | Set up by `has_rich_text :content` in `RichTextContent`; canonical for migrated articles |
| Legacy Markdown body | `articles.legacy_markdown_content` (`text`) | Fallback for pre-ActionText rows; `migrated_content?` selects the reader |
| Article metadata (`title`, `intro`, `price`, `asset_id`, `state`, `*_revenue_ratio`, …) | `articles` table (PostgreSQL) | The `Article` model itself |
| Content history | `article_snapshots.raw` (PostgreSQL JSON) | One row per save touching `title`, `intro`, `content`, or `published_at`; exposes `title`, `intro`, `content`, `digest` |
| Inline images | `active_storage_blobs` (`has_many_attached :images`) | Referenced from ActionText Trix content |
| Social-share poster | `active_storage_blobs` (`has_one_attached :poster`) | Generated by `Articles::GeneratePosterJob` on publish |
| Cover image | `active_storage_blobs` (`has_one_attached :cover`) | Optional, set by the author |

`Article#content` renders through `RichTextContent#content_as_html`: ActionText for migrated articles, otherwise `legacy_markdown_content` through `MarkdownRenderService`. `Article#content=` writes through ActionText, and the `after_save` hook calls `generate_snapshot` when `should_generate_snapshot?` is true.

## Publish flow boundaries

When an author runs `Article#publish!`, the row transitions to `published`, `notify_for_first_published` fires (see [Reference → Notifiers](../reference/notifiers.md#articlepublishednotifier)), and `Articles::GeneratePosterJob` enqueues a social-share poster render on the `low` queue.

That is the complete storage-related flow: no external upload job, no outbound call to a permanence network, and no background mirror. The poster job writes a derived Open Graph image back to ActiveStorage. Per-article Mixin wallets are also no longer provisioned on publish; settlement now routes to the author's Mixin identity (see [Value net → Settlement targets](./value-net.md#settlement-targets-post-1797)).

## Removed or unchanged surfaces

| Removed | Details |
|---------|---------|
| Arweave upload path | `app/jobs/articles/arweave_upload_job.rb`, related Mixin-bot deliveries, admin retry tooling |
| Arweave persistence | `app/models/arweave_transaction.rb` and the `arweave_transactions` table, dropped by `db/migrate/20260611052000_drop_arweave_transactions.rb` |
| External integration dependencies | The `arweave` and `graphql-client` gems |
| Product entry points | The author dashboard's **"Web3 Proof of Publishing"** panel and the Mirror.xyz import flow |

Historical migrations such as `db/migrate/20220810031609_create_arweave_transactions.rb` remain per Rails conventions but are inert on a fresh database. The publishing AASM state, `Article#may_publish?`, `ArticleSnapshot` creation, API reader path (`GET /api/articles/:uuid` through `RichTextContent#content_as_html`), and unrelated `solid_queue` / `solid_cable` infrastructure did not change.

## Capability guardrail and further reading

The `content-storage` capability says Quill stores published content in PostgreSQL and ActiveStorage and serves it without external lookups. Any future Arweave, Filecoin, IPFS, or similar mirror must update [`openspec/specs/content-storage/spec.md`](../../openspec/specs/content-storage/spec.md) first; the spec is the source of truth, not this page.

See also [Architecture → Persistence](./architecture.md#persistence), [Reference → Background jobs](../reference/background-jobs.md#articles), and [Value net](./value-net.md) for adjacent publishing, job, and payment flows.
