## Testing Notes

- **Framework**: Minitest (~> 5.25) with Capybara for system tests
- **Test location**: `test/` mirrors `app/` structure; `test/services/<namespace>/` for service objects (e.g. `test/services/orders/distribute_service_collection_test.rb`)
- **Test naming**: `*_test.rb` pattern
- **Fixtures**: YAML fixtures in `test/fixtures/`
- **Helpers**: CommerceHelpers, NotifierHelpers, QuillBotStub available in all tests
- **ActiveJob**: Test helper `perform_all_jobs` available
- **Auth**: `sign_in(user)` helper creates Session for web tests; `api_headers(access_token)` for API tests
- **Tests cannot run locally** due to Arweave GraphQL API network restrictions in sandbox â€” but other tests run fine on local Postgres
- **MarkdownRenderService / RichTextRenderService**: Pure services (no DB). `FastImage.size` may return nil for unreachable URLs; tests assert on wrapping behavior, not on width/height. iframe-src whitelist = YouTube only.
- **PreOrder**: AASM state machine has `drafted â†’ paid` and `drafted â†’ expired` events. `pay!` fires `broadcast_to_views` as after_commit (stub with `define_singleton_method(:broadcast_to_views) { }` in tests). `setup_attributes` callback auto-fills `follow_id`, `trace_id` (via `item.payment_trace_id payer`), `memo` (urlsafe base64, no padding, no `+`/`/`), `payee_id` (= QuillBot.client_id), and `asset_id` (from item).
- **PreOrder decoded_memo keys**: `t` (BUY/REWARD), `a` (article uuid), `l` (collection uuid for buy_collection), `f` (follow_id).
- **`safeoutputs create_pull_request`**: Returns a patch/bundle path when the bridge is in patch mode rather than opening a real PR on the remote. The PR may not appear in `list_pull_requests` until the bridge pushes the branch. The 2026-06-07 run's PR #1542 was opened and later closed (not merged). The 2026-06-08 run's PR #1545 was opened and merged. The 2026-06-09, 2026-06-11, and 2026-06-12 runs all returned patch mode (no visible PR yet).
- **Articles::Purchasable concern**: `authorized?(user)` returns true if `(published? && free?) || author == user`, then `user.blank?` short-circuits to false. `may_buy_by?(user)` checks `author.block_user?(user)`, `user&.block_user?(author)`, then returns `published?`. The policy layer (`ArticlePolicy`) wraps these with `user.present?` checks.
- **block_user / block_user?**: `action_store :block, :user` from `action-store` gem provides these methods. The user can call `.block_user(other_user)` and `.block_user?(other_user)` to manage blocks.
- **action_store gem**: `subscribe_user_ids` and other `_ids` methods on User are auto-generated by `action_store` from the `has_many :subscribe_users, through: :subscribe_user_actions` association â€” they are NOT defined in the User model itself.
- **Orders::DistributeService#distribute_collection_order!**: Creates only `quill_revenue` (conditional on `payment.wallet_id != QuillBot.api.client_id`) and `author_revenue`. `author_revenue.amount = (total - quill_amount).floor(8)`. `author_revenue.memo = "{buyer.name} bought {item.name}".truncate(70)`. `author_revenue.opponent_id = item.author.mixin_uuid`. `author_revenue.trace_id = MixinBot::Utils.unique_uuid(trace_id, item.author.mixin_uuid)`. NO reader_revenue, NO reference_revenue for buy_collection. `revenue_asset_id = payment.swap_order&.fill_asset_id || payment.asset_id`.
- **MixinNetworkSnapshot.create!**: Requires `transferred_at` field, otherwise validation fails. In tests, set `transferred_at: Time.current`.
- **`payment.wallet_id` in tests**: Returns `snapshot&.user_id`. Without a snapshot bound, it's nil. To test the `payment.wallet_id == QuillBot.api.client_id` branch, create a real `MixinNetworkSnapshot` with `user_id: bot_client_id`.
