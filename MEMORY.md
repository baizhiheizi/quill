## Testing Notes

- **Framework**: Minitest (~> 6.0, locked to 6.0.6) with Capybara for system tests
- **Test location**: `test/` mirrors `app/` structure; `test/services/<namespace>/` for service objects (e.g. `test/services/orders/distribute_service_collection_test.rb`)
- **Test naming**: `*_test.rb` pattern
- **Fixtures**: YAML fixtures in `test/fixtures/`
- **Helpers**: CommerceHelpers, NotifierHelpers, QuillBotStub available in all tests
- **ActiveJob**: Test helper `perform_all_jobs` available
- **Auth**: `sign_in(user)` helper creates Session for web tests; `api_headers(access_token)` for API tests
- **PostgreSQL required for tests**; sandbox has no local Postgres so `bin/rails test` cannot be run locally — rely on CI's Postgres service container
- **MarkdownRenderService / RichTextRenderService**: Pure services (no DB). `FastImage.size` may return nil for unreachable URLs; tests assert on wrapping behavior, not on width/height. iframe-src whitelist = YouTube only.
- **PreOrder**: AASM state machine has `drafted → paid` and `drafted → expired` events. `pay!` fires `broadcast_to_views` as after_commit (stub with `define_singleton_method(:broadcast_to_views) { }` in tests). `setup_attributes` callback auto-fills `follow_id`, `trace_id` (via `item.payment_trace_id payer`), `memo` (urlsafe base64, no padding, no `+`/`/`), `payee_id` (= QuillBot.client_id), and `asset_id` (from item).
- **PreOrder decoded_memo keys**: `t` (BUY/REWARD), `a` (article uuid), `l` (collection uuid for buy_collection), `f` (follow_id).
- **`safeoutputs create_pull_request`**: Returns a patch/bundle path when the bridge is in patch mode rather than opening a real PR on the remote. The PR may not appear in `list_pull_requests` until the bridge pushes the branch. The 2026-06-08 run's PR #1545 was opened and merged. The 2026-06-09, 06-11, 06-12, and 06-14 runs all returned patch mode (no visible PR yet).
- **Articles::Purchasable concern**: `authorized?(user)` returns true if `(published? && free?) || author == user`, then `user.blank?` short-circuits to false. `may_buy_by?(user)` checks `author.block_user?(user)`, `user&.block_user?(author)`, then returns `published?`. The policy layer (`ArticlePolicy`) wraps these with `user.present?` checks.
- **block_user / block_user?**: `action_store :block, :user` from `action-store` gem provides these methods.
- **action_store gem**: `subscribe_user_ids` and other `_ids` methods on User are auto-generated by `action_store`.
- **Orders::DistributeService#distribute_collection_order!**: Creates only `quill_revenue` (conditional on `payment.wallet_id != QuillBot.api.client_id`) and `author_revenue`. `author_revenue.amount = (total - quill_amount).floor(8)`. `author_revenue.memo = "{buyer.name} bought {item.name}".truncate(70)`. `author_revenue.opponent_id = item.author.mixin_uuid`. `author_revenue.trace_id = MixinBot::Utils.unique_uuid(trace_id, item.author.mixin_uuid)`. NO reader_revenue, NO reference_revenue for buy_collection.
- **Orders::DistributeService#distribute_article_order!**: Creates `quill_revenue` (conditional), `reader_revenue` (per early reader), `reference_revenue` (per `CiterReference`, Base64 memo `{t:"CITE",a:ref_uuid,c:item_uuid}`), `reader_revenue` for collection revenue (`"collection revenue from {title}".truncate(70)` memo, equal share per subscriber), and `author_revenue`. Memo formats: reader=`"Reader revenue from {title}".truncate(70)`, author=`"{buyer} {buy_article? ? 'bought' : 'rewarded'} {title}"` or `"Reference revenue from {title}"` for `cite_article?`. Reference uses `find_or_create_by` (not `!`) for idempotency. `distributor_wallet_id = QuillBot.api.client_id`.
- **MixinNetworkSnapshot.create!**: Requires `transferred_at` field. To test `payment.wallet_id == QuillBot.api.client_id` branch, create a real `MixinNetworkSnapshot` with `user_id: bot_client_id` and re-bind payment's snapshot_id.
- **Order type enum**: `buy_article: 0, reward_article: 1, cite_article: 2, buy_collection: 3`. cite_article orders have `citer` set to the citing article.
